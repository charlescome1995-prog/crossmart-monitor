Opening hook to write:

- Start with a concrete operator/developer pain point.
- Avoid sounding like a product ad in the first third.
- Use CrossMart as the practical example, not the whole story.

## Dashboards are not the goal

- Operators do not want more numbers.
- They want to know what to watch, what to change, and what to ignore.

## Start from repeated questions

- What changed?
- What product should I consider?
- How should I write this listing?
- Why is this competitor moving?

## Small tools beat one giant tool

- Monitor, Selector, Listing, Ops each answer a different question.
- Shared data patterns keep the suite coherent.

## Designing for decisions

- Score, filter, highlight, explain.
- Show evidence before recommendation.

## Hard lesson

- If a page cannot explain its recommendation, users will treat it like decoration.

## What I would improve next

- Add confidence levels.
- Add operator notes.
- Add feedback loops from user decisions.
